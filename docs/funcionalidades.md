#  Funcionalidades Principais

## 3.1 Requisitos Funcionais (RF)

**RF01** – O sistema deve permitir cadastro de pacientes.  
**RF02** – O sistema deve permitir cadastro de médicos e especialidades.  
**RF03** – O paciente deve conseguir agendar consultas.  
**RF04** – O médico deve registrar evolução médica no prontuário.  
**RF05** – O sistema deve emitir relatórios administrativos.  
**RF06** – O paciente deve visualizar exames e histórico médico.  

---

## 3.2 Requisitos Não Funcionais (RNF)

**RNF01** – O sistema deve garantir segurança e criptografia dos dados (LGPD).  
**RNF02** – A interface deve ser simples e responsiva.  
**RNF03** – O tempo de resposta deve ser inferior a 2 segundos.  
**RNF04** – O sistema deve estar disponível 99% do tempo.  
**RNF05** – O login deve exigir autenticação segura.
